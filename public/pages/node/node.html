Introduction

Node.js is an open-source and cross-platform runtime environment built on Chrome’s V8 JavaScript engine for executing JavaScript code outside of a browser. You need to recollect that NodeJS isn’t a framework, and it’s not a programing language. It provides an event-driven, non-blocking (asynchronous) I/O and cross-platform runtime environment for building highly scalable server-side applications using JavaScript.

Why node.js?

There could be several reasons to choose node.js. Here are the top reasons which are listed below.
  • Open-source and cross-platform
  • Easy to scale up in the future
  • Easy to learn
  • Huge community support
  • Fast
  • Realtime web apps

Node package manager(npm)

Node package manager is the default package manager for the javascript runtime environment node.js.It consists of a command line client, also called npm, and an online database of public and paid-for private packages, called the npm registry. The registry is accessed via the client, and the available packages can be browsed and searched via the npm website. The package manager and the registry are managed by npm, Inc.Which is written in JavaScript.

Package.json

The package.json is a json file that all npm packages contains, usually in the root directory.
The file holds metadata relevant to the project.
This could be the project name, version, needed dependencies and so on.

Following the JSON data format the fields are divided into key:value pairs,
with the possibility for objects, like the dependencies object, to hold multiple key:value pairs. 
            
                {
                    "name": "nodejs-note",
                    "version": "1.0.0",
                    "description": "This is the mandatory assignment project in fourth semester2022",
                    "main": "app.js",
                    "scripts": {
                      "test": "echo \"Error: no test specified\" && exit 1"
                    },
                    "keywords": [
                      "note",
                      "node",
                      "express"
                    ],
                    "author": "Krishna Prasad Khanal",
                    "license": "MIT",
                    "dependencies": {
                      "express": "^4.17.3",
                      "g": "^2.0.1",
                      "nodemon": "^2.0.15"
                    }
                  }

npm init

npm init command can be used to create a package.json file from scratch, if no such file exists.

When running the command with no arguments, a short guide appears asking you questions to help populate the resulting package.json.
As an example you will be asked to name the package, provide a version (1.0.0 by default), provide an url to the git repository (if any) etc.
